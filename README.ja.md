## IotaPolisを選ぶ理由

従来のフォーラムは中央サーバーに依存しており、停止・検閲・侵害のリスクがあります。**IotaPolis**はすべてのデータをIOTA 2.0ブロックチェーン上にMoveスマートコントラクトのイベントとして保存します。ローカルサーバーは単なるキャッシュであり、ブロックチェーンが唯一の信頼源です。

- **真の分散化** — 各ユーザーは独自のIOTAウォレット（Ed25519）を持ちます。サーバーは秘密鍵を一切保持しません
- **単一障害点なし** — どのノードからでもオンチェーンイベントからフォーラム全体を再構築可能
- **不変の履歴** — すべての投稿、編集、投票はトランザクションダイジェストとともに永続的に記録
- **オンチェーン権限管理** — ロール（ユーザー、モデレーター、管理者）はサーバーではなくスマートコントラクトが管理
- **組み込みのエコノミー** — チップ、サブスクリプション、有料コンテンツ、バッジ、エスクロー — すべてオンチェーン
- **テストネットは手数料ゼロ** — IOTA 2.0 Rebasedテストネットは自動Faucetで無料ガスを提供

## クイックスタート

```bash
# クローン
git clone https://github.com/deduzzo/iotapolis.git
cd iotapolis

# 依存関係のインストール
npm install
cd frontend && npm install && cd ..

# 初回起動 — サーバーウォレットと設定を生成
npm run dev
# "Sails lifted"が表示されたら Ctrl+C

# MoveスマートコントラクトをIOTAテストネットにデプロイ
npm run move:deploy

# フォーラムを起動
npm run dev
```

`http://localhost:5173` を開き、ウォレットを作成し、Faucetからガスを取得し、ユーザー名を登録して投稿を開始しましょう。

> 本番デプロイ、カスタムネットワーク、高度な設定については [DEPLOY.md](DEPLOY.md) を参照してください。

## アーキテクチャ

```
ブラウザ (React 19 + Vite 6 + TailwindCSS 4)
  |
  |-- IOTAのEd25519ウォレット（ニーモニック派生、localStorageにAES暗号化保存）
  |-- ブロックチェーン上で直接トランザクションに署名・実行
  |-- リッチWYSIWYGエディター (Tiptap) -> markdown
  |-- テーマエンジン（7プリセット、CSS変数）
  |-- ウォレットページ：残高、チップ、サブスクリプション、エスクロー
  |
  |  REST API（読み取り専用キャッシュ）+ Socket.io WebSocket
  v
サーバー (Sails.js + Node.js) — インデクサー専用
  |
  |-- ブロックチェーンイベントをSQLiteキャッシュにインデックス
  |-- Faucet：新規ユーザーにガスを送金（テストネット）
  |-- 高速クエリ用にキャッシュデータをREST経由で提供
  |-- 状態変更ごとにWebSocketブロードキャスト
  |-- クロスノード同期用の30秒ブロックチェーンポーリング
  |-- ユーザーの代わりにトランザクションの署名・発行はしない
  |
  v
Moveスマートコントラクト（オンチェーン、不変）
  |
  |-- Forum（共有）：ユーザーレジストリ、ロール、サブスクリプション、バッジ、レピュテーション、トレジャリー
  |-- Escrow（共有オブジェクト）：マルチシグ2-of-3資金管理
  |-- AdminCap（所有）：デプロイヤー権限
  |-- ロールゲート付き20以上のエントリー関数
  |-- すべての操作でイベントを発行（gzip圧縮JSONペイロード）
  |-- すべての決済を処理：チップ、サブスクリプション、購入、エスクロー
  |
  v
IOTA 2.0 Rebased（信頼の源泉）
  |
  |-- Package IDでイベント照会可能
  |-- すべてのノードが同じデータを参照
  |-- テストネットは手数料ゼロ
```

### データフロー

```
ユーザーが投稿を入力
  -> Tiptapエディターがmarkdownにシリアライズ
  -> フロントエンドがJSONペイロードをgzip圧縮
  -> ユーザーのEd25519キーペアがトランザクションに署名
  -> トランザクションがIOTAブロックチェーン上で直接実行
  -> スマートコントラクトがロールを確認（USER >= 1）、ForumEventを発行
  -> バックエンドがポーリング/購読でイベントを検出
  -> ローカルSQLiteキャッシュを更新
  -> WebSocket経由で'dataChanged'をブロードキャスト
  -> 接続中のすべてのクライアントがUIを更新
```

## スマートコントラクト

Moveスマートコントラクト（`move/forum/sources/forum.move`）はセキュリティの要です。すべての権限と決済はサーバーではなくIOTAバリデーターが管理します。

### ロールシステム

| レベル | ロール | 権限 |
|--------|--------|------|
| 0 | **BANNED** | すべての操作がバリデーターにより拒否 |
| 1 | **USER** | 投稿、返信、投票、自コンテンツ編集、チップ、サブスクリプション、購入 |
| 2 | **MODERATOR** | + カテゴリ作成、コンテンツモデレーション、BAN/解除、エスクロー仲裁 |
| 3 | **ADMIN** | + フォーラム設定、ロール管理、階層/バッジ設定、トレジャリー引き出し |

### エントリー関数

**フォーラム（基本）：**

| 関数 | 最低ロール | 用途 |
|------|-----------|------|
| `register()` | なし | 一回限りの登録、ROLE_USERを付与 |
| `post_event()` | USER | スレッド、投稿、返信、投票 |
| `mod_post_event()` | MODERATOR | カテゴリ、モデレーションアクション |
| `admin_post_event()` | ADMIN | フォーラム設定、ロール変更 |
| `set_user_role()` | MODERATOR | ユーザーロール変更（制約あり） |

**決済：**

| 関数 | 最低ロール | 用途 |
|------|-----------|------|
| `tip()` | USER | 投稿者にIOTAを送金 |
| `subscribe()` | USER | 階層プランに登録 |
| `renew_subscription()` | USER | 既存サブスクリプションの更新 |
| `purchase_content()` | USER | 有料コンテンツへのアクセスを購入 |
| `purchase_badge()` | USER | バッジを購入 |
| `configure_tier()` | ADMIN | サブスクリプション階層の追加/編集 |
| `configure_badge()` | ADMIN | バッジの追加/編集 |
| `withdraw_funds()` | ADMIN | フォーラムトレジャリーからの引き出し |

**エスクロー：**

| 関数 | 実行者 | 用途 |
|------|--------|------|
| `create_escrow()` | 買い手 | エスクローに資金をロック（2-of-3マルチシグ） |
| `mark_delivered()` | 売り手 | サービスの納品完了をマーク |
| `open_dispute()` | 買い手 | 紛争を開始 |
| `vote_release()` | 任意の当事者 | 売り手への資金リリースに投票 |
| `vote_refund()` | 任意の当事者 | 買い手への返金に投票 |
| `rate_trade()` | 買い手/売り手 | 相手方を評価（1-5つ星） |

### セキュリティ

- 各ユーザーは自身のEd25519キーペアで署名 — `ctx.sender()`はIOTAバリデーターが検証
- サーバーはユーザーの秘密鍵を一切保持しない
- エスクローはクロスバリデーション付き2-of-3投票（両側への投票は不可）
- 過払いは自動的に返金（おつりを正確に返却）
- エスクロー操作の期限管理
- BANされたユーザーはコントラクトレベルで拒否
- 自分より上のロールへの昇格不可、同等以上のロールのユーザーの変更不可

## 決済とマーケットプレイス

### チップ

任意の投稿のチップボタンをクリックして、著者にIOTAを直接送金できます。プリセット金額（0.1、0.5、1.0 IOTA）またはカスタム金額から選択。チップは即時、オンチェーン、仲介者ゼロです。

### サブスクリプション

管理者が価格と期間付きのサブスクリプション階層を設定します。ユーザーは階層の価格を支払って登録します。スマートコントラクトが有効期限とアクセス制御を自動管理します。

### 有料コンテンツ

著者はスレッドに価格を設定できます。コンテンツはAES-256で暗号化されます。支払い（オンチェーン）後、購入者は復号鍵を受け取ります。5%の手数料がフォーラムトレジャリーに送られます。

### エスクロー

ユーザー間のサービスでは、買い手がオンチェーンエスクローに資金をロックします。3者（買い手、売り手、仲裁者）が2-of-3マルチシグを構成。任意の2者が資金のリリースまたは返金を決定できます。解決時に2%の手数料がフォーラムトレジャリーに送られます。

### レピュテーション

エスクロー解決後、両当事者が評価（1-5つ星＋コメント）を残せます。評価はオンチェーンで不変です。ユーザープロフィールには平均評価、取引数、成功率、取引量が表示されます。

## マルチノード

IotaPolisは同一スマートコントラクトに接続する複数の独立ノードをサポートします。各ノードは：

1. 独自のSails.jsサーバー＋Reactフロントエンドを実行
2. 独自のSQLiteキャッシュを保持（再構築可能）
3. ユーザーがオンチェーンで直接トランザクションに署名
4. 30秒ごとにブロックチェーンから同期

### 既存フォーラムへの参加

```bash
# サーバーを起動
npm run dev

# ブラウザで：Setup -> "Connect to existing forum"に移動
# 接続文字列を貼り付け：testnet:0xPACKAGE_ID:0xFORUM_OBJECT_ID
# システムがブロックチェーンからすべてのイベントを同期
```

| POST | `/api/v1/faucet-request` | 新規アドレスにガスを要求（レート制限あり） |
| POST | `/api/v1/full-reset` | 完全リセット（管理者署名が必要） |
| POST | `/api/v1/sync-reset` | キャッシュリセット＋再同期（管理者署名が必要） |

すべての書き込み操作（投稿、投票、モデレーション、決済、エスクロー）はユーザーのウォレットからIOTAブロックチェーン上で直接実行されます。サーバーは読み取り専用のインデクサーです。

---

## ID（アイデンティティ）の仕組み

1. **生成** — ブラウザがBIP39ニーモニック（12単語）からEd25519キーペアを作成
2. **暗号化** — ニーモニックをユーザーのパスワードで暗号化（AES-256-GCM + PBKDF2）しlocalStorageに保存
3. **Faucet** — バックエンドが新規アドレスにガスIOTAを送金（テストネット）
4. **登録** — ユーザーがMoveコントラクトの`register()`を直接呼び出し
5. **署名** — すべてのアクション（投稿、投票、チップ、エスクロー）はユーザーのEd25519鍵で署名されたトランザクション
6. **検証** — `ctx.sender()`はプロトコルレベルでIOTAバリデーターが検証
7. **バックアップ** — ユーザーは12単語のニーモニックをエクスポートして任意のデバイスで復元可能

サーバーにパスワードなし。メールなし。アカウントなし。あなたのウォレットがあなたのIDです。
